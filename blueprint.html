Core Fantasy

Monster breeding, training, and battling - Fast-paced, real-time action where the player acts as commander/coach of their monster team.

2.2 Generation System

Each species has 4 generations of monsters:

GENERATION STRUCTURE (Per Species)
==================================

Gen 1: 4 Starter Types (A, B, C, D)
         ↓ Breed within Gen 1
Gen 2: 6 Combinations (AB, AC, AD, BC, BD, CD)
         ↓ Breed within Gen 2
Gen 3: 15 Combinations (all pairs of Gen 2)
         ↓ Cross-species breeding (secret)
Gen 4: Special hybrids (cross-species)

TOTAL PER SPECIES: 4 + 6 + 15 = 25 monsters
TOTAL ALL SPECIES: 25 × 4 = 100 monsters
SECRET GEN 4: 6+ additional cross-species monsters
GRAND TOTAL: 106+ unique monster types

3. BREEDING SYSTEM

3.1 Breeding Rules

Core Rules

  1. Same Generation Only: Monsters can only breed with others of the same generation
  2. Same Species Only (Except Gen 4): Gen 1-3 breeding stays within species
  3. Age Requirement: Monsters must reach [EXAMPLE: Age 30 days] before breeding eligibility
  4. Cooldown: After breeding, a monster cannot breed again for [EXAMPLE: 7 days]
  5. Fusion Alternative: Instead of breeding, two monsters can fuse (both are consumed, offspring is stronger)

Breeding vs Fusion

Aspect Breeding Fusion
Parents afterBoth surviveBoth consumed
Offspring statsAverage of parentsHigher than average
Offspring levelStarts at 1Starts at [EXAMPLE: 10]
Inherited movesShared moves onlyAll moves from both
CooldownYesN/A (parents gone)

3.2 Inheritance Formulas

Stat Inheritance

OFFSPRING STAT CALCULATION [EXAMPLE FORMULA]
=============================================

For each stat (STR, INT, AGI, END):

Base = (Parent1_Stat + Parent2_Stat) / 2
Variance = Random(-2, +2)
Birthmark_Bonus = Apply birthmark modifier

Final_Stat = Base + Variance + Birthmark_Bonus

FUSION BONUS: Add +15% to Base before variance

Move Inheritance

MOVE INHERITANCE RULES
======================

1. List all moves Parent 1 knows
2. List all moves Parent 2 knows
3. Find moves that appear in BOTH lists AND offspring can learn
4. Offspring starts with those shared learnable moves
5. Remaining move slots are empty

FUSION: Offspring inherits ALL moves from both parents that it can learn

4. COMBAT SYSTEM

4.1 Battle Overview

  • 100% Real-time combat (not turn-based)
  • Player controls 2 monsters simultaneously
  • Player gives orders but monsters have autonomy based on INT and Temperament
  • Battle formats: 1v1, 1v2, 2v2

4.3 Battle Flow

Option A: Step-by-Step Flow

BATTLE FLOW - STEP BY STEP
==========================

1. BATTLE START
   ├── Camera zooms to battle arena
   ├── Monsters take starting positions
   └── "BATTLE START" notification

2. REAL-TIME COMBAT LOOP
   ├── Monsters act based on current STANCE
   ├── Player can give ORDERS at any time
   ├── Monsters respond based on INT + Temperament
   ├── Attacks connect and deal damage
   ├── Energy regenerates over time
   └── Loop continues until victory condition

3. VICTORY CONDITIONS
   ├── All enemy monsters HP = 0 → WIN
   ├── All player monsters HP = 0 → LOSE
   └── Timer expires (arena only) → Judge decision

4. BATTLE END
   ├── XP awarded based on performance
   ├── Stats increase based on actions taken
   ├── Temperament adjusted
   └── Return to field/arena

Combined Formula

ORDER SUCCESS CALCULATION
=========================

Base_Success = 50%
INT_Bonus = INT × 2%
Temp_Bonus = Temperament × 0.3%
Stance_Match = +20% if order matches current stance

Total = Base_Success + INT_Bonus + Temp_Bonus + Stance_Match

Example:
  Monster with INT 12, Temperament 60
  Base: 50%
  INT: +24%
  Temp: +18%
  Total: 92% success rate

If failed: Monster does its own thing based on stance
